# TeamPro 教練戰情室

> 讓教練每天 3 分鐘掌握整隊狀態。
> 多租戶訂閱制 SaaS（GAS + Google Sheet + 純前端四檔），第一階段手動開通付費方案、不串金流。

## 檔案結構

| 檔案 | 角色 | 說明 |
|---|---|---|
| `index.html` | 公開 | 銷售首頁（痛點/解決方案/功能/方案/FAQ） |
| `app.html` | 教練 | 註冊登入、今日戰情室、選手管理、團隊與連結 |
| `join.html` | 選手 | 透過團隊連結填寫每日 30 項 KPI + 身體/飲食/心情 |
| `admin.html` | 管理者 | 搜尋教練、改方案、延期、停用、營運統計 |
| `api.js` | 共用 | API 呼叫 / session / KPI 題庫 / UI 工具 |
| `style.css` | 共用 | 樣式 |
| `apps-script/Code.gs` | 後端 | 多租戶 GAS Web App |

## 部署步驟

### 1. 後端（GAS）
1. 建立一份新的 Google 試算表 →「擴充功能 > Apps Script」。
2. 把 `apps-script/Code.gs` 全部貼上、儲存。
3. 編輯器中執行一次 **`setup()`**（授權並建立 coaches/sessions/teams/athletes/records/audit 六個分頁）。
4. 設定管理者密碼：把 `setAdminPassword()` 內字串改成你的密碼 → 執行一次 → **再把字串改回佔位字並儲存**（避免密碼留在程式碼）。
5. 「部署 > 新增部署 > 網頁應用程式」：執行身分＝**我**，存取權＝**任何人**。複製 `/exec` 網址。

> ⚠️ 改 `Code.gs` 後要讓變更生效，務必「**管理部署 > 編輯 > 版本：新版本**」，不要只存檔。

### 2. 前端
- 把 `index/app/join/admin.html`、`api.js`、`style.css` 放到同一個目錄（GitHub Pages 或任何靜態空間）。
- 建議：部署後把 `/exec` 網址填進 `api.js` 的 `DEFAULT_URL`，使用者就不必自己貼。
- 或讓教練在 `app.html` 的「系統設定」貼一次後端網址（存在瀏覽器 localStorage）。

## 使用流程

1. **教練**：`app.html?signup=1` 註冊 → 系統自動建「示範隊＋3 位假選手」立刻看到戰情室。
2. 建立自己的團隊 → 複製專屬填寫連結（已內含後端網址）。
3. 新增選手（免費版上限 5 人，超過會擋並提示升級）。
4. **選手**：開連結 → 選自己 → 勾同意 → 30 項滑桿（可「帶入上次」）→ 身體/心情 → 送出，即得 AI 飲食建議與分享文字。
5. **教練戰情室**：今日完成率、未繳名單（一鍵複製催繳）、紅黃綠燈、家長通知文字。
6. **管理者**：`admin.html` 登入 → 搜尋教練 → 改方案/設到期日/停用。

## KPI 量表（6 面向 × 5 項＝30 項，1–5 分，5＝最好）

- **技術**：動作準確度・動作穩定度・速度與反應・力量傳遞・技術完成度
- **戰術**：距離控制・出手時機・攻防轉換・對手判讀・教練戰術執行
- **體能**：爆發力・肌力・肌耐力・心肺耐力・敏捷與協調
- **心理**：專注力・壓力穩定・自信心・挫折恢復・訓練動機
- **態度**：準時與紀律・訓練投入・主動修正・接受指導・團隊合作
- **生理**：睡眠恢復・精神恢復・肌肉舒適度・傷勢安全度・整體恢復感

每面向＝5 項平均；總 KPI＝6 面向等權平均。燈號：≥4.0 綠、3.0–3.9 黃、<3.0 紅。

## 安全與隔離

- 教練身分一律由 session token 反推 `coachId`，所有查詢以 `coachId` 過濾（租戶隔離）。
- 密碼以 salt + 多輪 SHA-256 雜湊存放（GAS 無 bcrypt 的折衷）。
- 團隊連結 `shareToken` 可一鍵重設、舊連結即失效；教練停用則整團失效。
- 選手填寫前需勾選同意（含未成年家長同意聲明）。
